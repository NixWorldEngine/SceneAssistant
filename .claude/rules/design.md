# Design 文件夹规则 — 定稿输出规范

## 文件夹性质

`design/` 是最终设定集的存放处。所有写入此处的内容都应是经过用户确认的、完整的、可直接用于剧本创作的定稿。

## 目录结构

```
design/
├── character/          # 人物设定
├── world/              # 世界观设定
├── event/              # 事件设定
├── faction/            # 势力/组织设定
├── item/               # 物品/道具设定
├── relation/           # 关系图谱
└── timeline/           # 时间线总览
```

> 写入文件前，如果目标子目录不存在，应自动创建。

## 写入前置条件

**必须同时满足以下条件才可写入 design：**
1. 全局规则中对应类别的质量标准已全部达标
2. 所有 `[推测]` 标记的内容已经用户确认或删除
3. 已检查与 design 中现有文件的一致性，无冲突

## 文件格式规范

所有定稿使用 Markdown 格式，文件头部必须包含元信息块：

### 人物设定模板

```markdown
# {角色名}

> 来源：reference/{来源文件路径}
> 状态：定稿 | 修订日期：{YYYY-MM-DD}

## 基本信息
- 姓名：
- 性别：
- 年龄：
- 身份/职业：

## 外貌
<!-- 身高体型、五官特征、显著标记 -->

## 穿衣风格
<!-- 日常穿搭、特殊场合着装、配饰偏好 -->

## 说话风格
<!-- 语气、口头禅、用词习惯、语速节奏，附 1-2 句示例台词 -->

## 性格
<!-- 核心性格、性格矛盾面、压力下的表现 -->

## 人物经历
<!-- 按时间线梳理：童年 → 转折点 → 当前处境 -->

## 社会关系
<!-- 家庭、友人、敌对、情感线，标注与其他已有角色的关联 -->

## 备注
<!-- 设计意图、角色弧线方向、创作时的注意事项 -->
```

## 修订规则

- 修改已有定稿时，更新文件头部的修订日期
- 如果修改涉及与其他文件的关联（如角色关系变动），需同步检查并更新关联文件
- 重大修改前应告知用户影响范围

## 一致性检查

每次写入或修改 design 文件时，执行以下检查：
1. **人名一致性**：同一角色在所有文件中名称一致
2. **时间线一致性**：事件先后顺序无矛盾
3. **关系一致性**：A→B 的关系描述与 B→A 的描述吻合
4. **世界观一致性**：角色行为、事件发展符合世界观设定的规则
5. **引用有效性**：文件间的交叉引用指向实际存在的文件

> 如发现不一致，暂停写入并向用户报告。
