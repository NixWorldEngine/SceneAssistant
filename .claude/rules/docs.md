# OCS 接口文档

## 目录职责

`views/docs/` 存放 OCS 桥接层释放给 iframe 视图的全部 API 接口文档

| 目录 | 职责 |
|------|------|
| views/docs/README.md | 按功能分类的接口索引 |
| views/docs/api/ | 接口的独立文档文件 |

## 文档格式要求

每个接口文档必须包含以下内容
- 函数签名 (完整的函数名 + 参数列表)
- 参数类型 (每个参数的 TS 类型)
- 参数说明 (每个参数的用途)
- 参数枚举值 (如果参数有固定取值范围)
- 返回值类型和结构
- 函数说明 (功能描述 调用时机 注意事项)
- 使用示例

## 工作规范

- 文档与 views/mock/api-specs.json 对应 修改文档时同步更新 specs
- 新增接口时同时创建文档和 mock 条目

## 禁止操作

- 不要删除或重命名已有文档 视图代码可能依赖文档中的接口路径
- 不要在文档中包含真实的 API 密钥或用户数据
