## 三、授权范围

### 3.1 测试范围

授权测试方对目标 **sexyai.top (192.144.189.8)** 的全部页面及相关服务进行安全评估，具体包括但不限于：

#### Web应用安全测试

- **跨站脚本攻击 (XSS)**
  - 反射型 XSS
  - 存储型 XSS
  - DOM 型 XSS
  - Self-XSS

- **跨站请求伪造 (CSRF)**
  - 表单提交 CSRF
  - JSON 请求 CSRF
  - Token 验证机制绕过

- **远程代码执行 (RCE)**
  - 命令注入
  - 代码注入
  - 反序列化漏洞
  - 模板注入 (SSTI)
  - 表达式注入

- **SQL 注入**
  - 联合查询注入
  - 布尔盲注
  - 时间盲注
  - 报错注入
  - 堆叠注入

- **文件相关漏洞**
  - 任意文件上传
  - 任意文件读取/下载
  - 文件包含（LFI/RFI）
  - 目录遍历

- **身份认证与授权**
  - 登录绕过
  - 越权访问（水平/垂直）
  - 会话管理缺陷
  - JWT/Token 安全
  - 密码重置逻辑缺陷

- **服务端请求伪造 (SSRF)**
  - 内网探测
  - 协议利用
  - 云元数据访问

- **业务逻辑漏洞**
  - 支付逻辑绕过
  - 验证码绕过
  - 竞争条件（Race Condition）
  - 参数篡改

- **信息泄露**
  - 敏感数据暴露
  - 错误信息泄露
  - 源码泄露
  - 接口文档泄露
  - .git/.svn/.env 等配置文件泄露

- **其他常见漏洞**
  - XXE（XML外部实体注入）
  - CORS 配置不当
  - 点击劫持 (Clickjacking)
  - HTTP 请求走私
  - WebSocket 安全
  - 开放重定向

### 3.2 基础设施测试

- 端口扫描与服务识别
- 中间件/CMS 版本探测与已知漏洞验证
- SSL/TLS 配置评估
- HTTP 安全头检查
- DNS 配置评估

### 3.3 明确排除范围

以下行为 **不在** 授权范围内：

- 对非目标IP和域名的任何测试
- 拒绝服务攻击 (DoS/DDoS)
- 社会工程学攻击
- 对第三方服务/CDN 的直接攻击
- 破坏、删除或篡改生产数据

## 五、测试规则与约束

1. 测试过程中如发现高危漏洞（如 RCE、SQL注入等），应 **第一时间** 通知授权方。
2. 禁止利用发现的漏洞获取超出验证目的的数据。
3. 测试过程中获取的所有数据、截图、日志等均属机密，不得向第三方泄露。
4. 如测试导致服务异常，应立即停止并通知授权方。
5. 测试工具和方法应控制在合理范围内，避免对业务造成不必要的影响。
6. 所有测试操作需保留完整日志记录。

## 六、交付物

测试完成后，测试方应交付以下内容：

1. **渗透测试报告** — 包含所有发现的漏洞详情、风险等级、复现步骤
2. **漏洞修复建议** — 针对每个漏洞提供具体修复方案
3. **测试日志** — 完整的测试操作记录
4. **复测报告**（如适用） — 对修复后的漏洞进行验证

---

## 七、风险等级定义

| 等级 | 说明 |
|------|------|
| 严重 | 可直接获取服务器权限、大量敏感数据泄露（RCE、SQL注入拖库等） |
| 高危 | 可获取敏感信息或影响大量用户（存储型XSS、越权访问等） |
| 中危 | 需要一定条件才能利用（CSRF、反射型XSS、信息泄露等） |
| 低危 | 影响较小或利用难度极高（点击劫持、安全头缺失等） |
| 信息 | 安全建议类，不构成直接威胁 |
