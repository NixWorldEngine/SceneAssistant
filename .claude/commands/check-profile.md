# 检查人设一致性

## 第一步 读取所有档案

使用 Glob 工具扫描以下目录中的所有 .md 文件
- `profile/`
- `reference/`

使用 Read 工具逐一读取所有扫描到的文件 记录每个文件的完整内容

## 第二步 提取关键信息

从所有文件中提取以下维度的信息并建立索引

- 角色名称与别名
- 年龄/生日/时间线
- 外貌描述
- 性格特征
- 背景故事与经历
- 角色间关系 (亲属/恋人/敌对/从属等)
- 世界观设定 (地理/历史/魔法体系/科技水平等)
- 阵营与势力归属
- 能力与限制

## 第三步 交叉比对

逐一检查以下类型的冲突

| 冲突类型 | 检查方式 |
|----------|----------|
| 年龄矛盾 | 同一角色在不同文件中年龄不一致 或与时间线矛盾 |
| 关系矛盾 | A文件说X是Y的朋友 B文件说X是Y的敌人 |
| 背景矛盾 | 同一事件在不同文件中描述不同 |
| 世界观矛盾 | 规则/设定在不同文件间冲突 |
| 外貌矛盾 | 同一角色的外貌描述不一致 |
| 能力矛盾 | 角色能力与世界观限制冲突 |
| 时间线矛盾 | 事件顺序或时间点不一致 |

## 第四步 逐一报告

对于每个发现的冲突 按以下格式呈现给用户

```
冲突 #{序号}: {冲突类型}

文件A: {文件路径}
> {相关引用}

文件B: {文件路径}
> {相关引用}

说明: {为什么这是一个冲突}

请选择:
1. 以文件A为准
2. 以文件B为准
3. 自定义修改
```

等待用户对每个冲突做出选择后再继续下一个

## 第五步 应用修改

根据用户的选择 使用 Edit 工具修改对应的 profile/ 文件
- 不修改 reference/ 目录中的文件
- 每次修改后向用户确认修改结果
- 全部完成后输出修改摘要
