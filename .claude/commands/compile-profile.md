# 编译人设档案

## 第一步 读取所有档案

使用 Glob 工具扫描 `profile/` 目录中的所有 .md 文件
使用 Read 工具逐一读取所有文件内容
记录每个文件的原始字数

## 第二步 翻译与优化

对每个档案文件执行以下处理

### 翻译为英文
将中文内容翻译为自然流畅的英文

### 针对 Gemini tokenizer 优化
- 散文描述转为结构化键值对格式
- 删除重复/冗余的描述
- 合并多处出现的相同信息
- 使用简洁措辞 避免不必要的修饰词
- 在不产生歧义的前提下使用缩写 (例如 approx. / w/ / b/c)
- 列表项去掉不必要的连接词

### 优化示例

优化前:
```
她是一个非常温柔善良的女孩 总是喜欢帮助别人
她有着一头乌黑亮丽的长发 眼睛是深棕色的
```

优化后:
```
Personality: gentle, kind, helpful
Hair: long, black
Eyes: dark brown
```

## 第三步 写入输出

使用 Bash 工具创建 `output/` 目录 (如不存在)
对每个处理完的文件 使用 Write 工具写入 `output/{原文件名}.md`
保持与 profile/ 中相同的文件名

## 第四步 报告

输出处理摘要

| 文件 | 原始字数 | 优化后词数 | 预估token节省 |
|------|----------|------------|---------------|
| {文件名} | {中文字数} | {英文词数} | {百分比} |

预估方法: 中文每字约1.5 token / 优化英文每词约1.3 token
