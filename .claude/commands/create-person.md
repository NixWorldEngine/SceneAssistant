# 深度角色创建

## 第一步 基本信息采访

依次提问 每次等待用户回答后再继续

1. 角色的全名是什么 有没有别名或昵称
2. 年龄/生日
3. 性别与性取向
4. 整体外貌 (身高/体型/肤色/发型/瞳色)
5. 日常穿着风格

## 第二步 性格采访

6. MBTI类型 或者用3-5个词描述核心性格
7. 最大的恐惧是什么
8. 最深的渴望是什么
9. 有什么怪癖或小习惯
10. 愤怒/悲伤/高兴时分别什么反应

## 第三步 背景采访

11. 家庭构成 (父母/兄弟姐妹/其他重要亲属)
12. 成长环境
13. 受过什么教育或训练
14. 人生中最重要的三个事件
15. 有没有不愿提及的过去

## 第四步 关系采访

16. 和故事中哪些角色有关系 分别是什么关系

使用 Glob 和 Read 扫描 `profile/` 中已有角色 列出已有角色供用户参照

17. 有没有暗恋/仇恨/崇拜的对象
18. 社交风格 (外向/内向/选择性社交)

## 第五步 表达方式采访

19. 说话时的语气和措辞习惯
20. 有没有口头禅
21. 肢体语言特点 (紧张时摸耳朵/开心时跺脚等)

## 第六步 成长方向采访

22. 角色在故事开始时的核心矛盾是什么
23. 你希望这个角色最终走向什么方向

## 第七步 生成档案

基于所有采访内容 生成完整角色档案

```
# {角色全名}

## 基本信息
- 全名:
- 别名:
- 年龄:
- 生日:
- 性别:
- 身高/体型:
- 外貌特征:
- 穿着风格:

## 性格
- 类型: {MBTI或特质描述}
- 核心特质:
- 恐惧:
- 渴望:
- 怪癖:
- 情绪反应模式:

## 背景
- 家庭:
- 成长环境:
- 教育/训练:
- 关键事件:
- 隐藏的过去:

## 关系网络
{与每个相关角色的关系描述}

## 表达方式
- 语气:
- 措辞习惯:
- 口头禅:
- 肢体语言:

## 能力与弱点
- 擅长:
- 不擅长:

## 角色弧线
- 起点: {故事开始时的状态}
- 核心矛盾:
- 成长方向:
```

## 第八步 一致性检查

使用 Read 读取 `profile/` 中所有已有档案
检查新角色与已有设定是否存在冲突
如有冲突 向用户报告并询问如何处理

## 第九步 确认并写入

向用户展示完整档案
确认后使用 Write 工具写入 `profile/character-{name}.md`
文件名使用小写英文 空格用连字符替代
